# 请封装一个截取页面跳转传递的参数的方法？

    const url = 'http://www.runoob.com/index.html?id=8&type=1&token=1'

    function getUrlPrams(url) {
        const arr = url.split('?')
        if (arr[0] === url) {
            return false
        }
        const prams = arr[1].split('&')
        let pramObj = {}
        prams.forEach(item => {
            const pram = item.split('=')
            pramObj[pram[0]] = pram[1]
        })
        return pramObj
    }

    const prams = getUrlPrams(url)
    console.log(prams, prams.id) // { id: '8', type: '1', token: '1' } '8'

案例解析

首先使用split('?')方法，用?字符将完整的url转成一个数组，数组第一项是不包含参数的url，数组第二项是所有的参数组成的字符串。然后再用split('&')将参数截取成一个数组，进行遍历，将遍历出来的每一项继续用split('=')转成数组，数组第一项是参数名，第二项是参数值。最后把参数封装到一个对象中管理，对象属性是参数名，属性值就是参数值。如果请求接口是需要该参数，直接读取该对象的对应属性即可。

说明

该方法目前使用的场景如下：1、在使用原生JavaScript开发页面时，在使用window.location.href、window.location.assign()、window.location.replace()跳转页面后，可以通过 window.location.href 获取页面跳转后的完整url，调用上述方法截取参数。或者通过window.location.search直接获取跳转当前页面传递的参数，调用上述方法截取参数。2、在开发H5页面与Android、iOS交互时，可以通过window.location.href等方法获取 Android、iOS 跳转H5页面时通过 url 传递的参数，调用上述方法截取参数。另外需要说明的是，如果参数中包含 =、&、?这类字符时，使用split() 方法会出错，需要灵活使用slice、substr、substring等方法截取参数。

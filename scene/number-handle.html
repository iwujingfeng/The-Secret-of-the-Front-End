# 数字处理

这部分内容涵盖了大部分数字处理的方法，通过归纳对比，进行各种应用场景的的分析。首先

# 浮点数字保留小数点后指定位数

// 方法一
const num = 5.5678;
const newNum = num.toFixed(2);
console.log(newNum, typeof newNum); // 5.57 string
console.log(parseFloat(newNum), typeof parseFloat(newNum)); // 5.57 'number'
// 方法二
const num1 = 5.5678;
const newNum1 = Math.round(num1 * 100) / 100;
console.log(newNum1, typeof newNum1); // 5.57 'number'
// 方法三
const num2 = 5.5678;
const str = num2.toString();
const newNum2 = str.substring(0, str.indexOf(".") + 3);
console.log(newNum2, typeof newNum2); // 5.56 string
// 方法四
var num3 = 5.5678;
var re = /([0-9]+\.[0-9]{2})[0-9]*/;
const newNum3 = num3.toString().replace(re, "$1");
console.log(newNum3, typeof newNum3); // 5.56 string

案例解析

方法一是最常用的方式，使用 toFixed() 方法使浮点数保留小数点后的指定为数。不过，需要注意的时，toFixed()会进行四舍五入处理，同时toFixed() 的返回值也不是 'number' 类型，而是 string 类型。如果需要使用 'number' 类型，需要使用 parseFloat() 方法将 string 类型转化成 'number' 类型。

方法二的处理过程是这样的，先执行 num1 \* 100，将数字处理为 556.78，然后执行 Math.round()方法，将数字四舍五入为整数 556，然后除以 100，达到最终效果。该方法的优点在于，处理过程并不会改变值的类型，始终为 'number' 类型。

方法三是将数字先转为 string 类型，然后应用 substring()方法，从索引 0 的位置开始到小数点的索引后移三位进行截取，这样就可以截取小数点后两位了。需要注意的是，该方法无法进行四舍五入处理，返回值也是string 类型。

方法四使用正则匹配进行处理，先定义正则表达式 re 来匹配整个数字 5.5678，([0-9]+\.[0-9]{2})是一个分组，用来匹配 5.56，[0-9]+ 匹配了 5，\.匹配了.，[0-9]{2}匹配两位任意数字 56，后面的[0-9]\*匹配了剩余部分 78。replace(re, "$1")方法中，$1 代表正则表达式中的第一个分组，也就是([0-9]+\.[0-9]{2})，匹配的就是 5.56，所以替换过程就是，第一个正则表达式分组匹配的 5.56 把整个 re 正则表达式匹配的 5.5678 进行了替换。当然，该方法，没有进行四舍五入。

# 非数值类型转化成数值类型

console.log(Number(true), Number(new Boolean(true))); // 1 1
console.log(Number(false), Number(new Boolean(false))); // 0 0
console.log(Number("123"), Number(new String("123"))); // 123 123
console.log(Number("0.11")); // 0.11
console.log(Number("")); // 0
console.log(Number(new Date())); // 1554467983692
console.log(Number(null), Number(undefined)); // 0 NaN

console.log(parseInt("")); // NaN
console.log(parseInt(".")); // NaN
console.log(parseInt("123")); // 123
console.log(parseInt("123.5")); // 123
console.log(parseFloat("123")); // 123
console.log(parseFloat("123.5")); // 123.5

案例解析

Number() 函数把对象的值转换为数值，但是直接传入字面量形式的数据，依然能够正确转换，那是因为 JS 引擎默认将字面量形式转换成了对象形式。可以发现，Number() 函数的主要用途是可以用来将布尔值转换成数值，这在做一些数据映射的时候有一定用处。如果要把字符串类型的数字转成数值类型，要转成整数，使用 parseInt()，而parseInt()之所以可以把由浮点数字组成的字符串转成整数，是因为.这个字符无法正确解析，所有后面的都会忽略。通过代码可以发现，空字符串""和"."解析时都是NaN。要转成浮点数字使用 parseFloat()。需要注意的是，parseInt()可以对八进制（0 开头）、十六机制（0x 开头）、十进制等多种进制的值进行解析，parseFloat()只对十进制值进行解析。

# 将浮点数字转化成整数

const num = 5.05678;
console.log(parseInt(num), typeof parseInt(num)); // 5 'number'
console.log(parseInt(0.0000006)); // 6
console.log(0.0000006); // 6e-7
console.log(parseInt("0.00000060")); // 0

console.log(Math.ceil(5.05)); // 6
console.log(Math.ceil(5.55)); // 6
console.log(Math.ceil("5.55")); // 6
console.log(Math.floor(5.05)); // 5
console.log(Math.floor(5.55)); // 5
console.log(Math.trunc(5.05)); // 5
console.log(Math.trunc(5.55)); // 5
console.log(Math.trunc(-5.55)); // -5
console.log(Math.round(5.05)); // 5
console.log(Math.round(5.55)); // 6
console.log(Math.ceil(0.0000006)); // 1
console.log(Math.floor(0.0000006)); // 0
console.log(Math.trunc(0.0000006)); // 0
console.log(Math.round(0.0000006)); // 0

const num = 5.56;
const num1 = 5.05;
console.log(num.toFixed(), num1.toFixed()); // 6 5
console.log(num.toPrecision(1), num1.toPrecision(1)); // 6 5

console.log("位运算", num | 0); // 5
console.log("位运算", num1 | 0); // 5

案例解析

通过案例的代码，parseInt()主要可以将浮点数字构成的 字符串类型 转成整数，但在处理 数值类型 时是不安全的。譬如将数值类型的 0.0000006 转成整数时，0.0000006 首先会以科学计数形式 6e-7 存在，parseInt()会将 6e-7 首先转成字符串，因为转换时无法识别 e,所以后面的都会忽略，最终的字符串其实是'6'，然后将'6'解析为整数型数值就是 6。所以，如果转换为整数前，数字是数值 'number' 类型，尽量使用下面的方法。

Math.ceil()会向上取整，整数永远大于原始的浮点数。Math.floor()会向下取整，整数永远小于原始的浮点数。ES6 中的 Math.trunc()只会返回小数点前的整数。Math.round()会根据小数点后面第一个浮点数字进行四舍五入。需要注意的是，在处理 0.0000006 这类浮点数中包含若干 0 的数字时，也会返回正确的值。另外，请注意 Math.ceil("5.55") 一行，可以发现，这类方法都可以对 字符串类型 进行正确的处理，结果与处理 数值类型 时一样。

toFixed() 方法同样可以取整，使用 toFixed()将数字四舍五入转换为一个整数，形式上和 Math.round()一样，但是toFixed()转换后的整数是字符串类型，这是最大的区别。toPrecision()可以将数字格式化为指定的长度，因此，可以传入整数部分的长度值，将浮点数格式化为只保留整数部分，需要注意的是，该方法会进行四舍五入，同时处理结果也是字符串类型。该方法的弊端在于，必须先知道整数部分到底是多少位 s 数，才能进行格式化。

最后一个方法，使用了位运算符 |，因为浮点数是不支持位运算，所以会先把浮点数转成整数再进行位运算，对比可以发现，处理结果和 Math.floor()向下取整是一样的。

# 判断是否为整数类型

// 取余运算符判断
const num = 5.56;
const num1 = 5;
console.log("取余运算符", typeof num === "number" && num % 1 === 0); // false
console.log("取余运算符", typeof num === "number" && num1 % 1 === 0); // true
console.log("取余运算符", "5.56" % 1 === 0); // false
console.log("取余运算符", "5" % 1 === 0); // true
console.log("取余运算符", "" % 1 === 0); // true
console.log("取余运算符", [] % 1 === 0); // true
console.log("取余运算符", true % 1 === 0); // true
console.log("取余运算符", false % 1 === 0); // true

// Math.round、Math.ceil、Math.floor和 Math.trunc 判断
console.log(Math.ceil(num) === num); // false
console.log(Math.ceil(num1) === num1); // true

// 通过parseInt判断
console.log(parseInt(num) === num); // false
console.log(parseInt(num1) === num1); // true
console.log(parseInt(1000000000000000000000) === 1000000000000000000000); // false
console.log(parseInt(0.0000006) === 0.0000006); // false

// 通过位运算判断
console.log("位运算"(num | 0) === num); // false
console.log("位运算"(num1 | 0) === num1); // true
console.log((Math.pow(2, 32) | 0) === Math.pow(2, 32)); // false

// Number.isInteger
console.log(Number.isInteger(num) === num); // false
console.log(Number.isInteger(num1) === num1); // true

案例解析

第一种方法是利用了取余运算符，依据的是任何整数都可以被 1 整除（余数是 0）这一个规则。通过判断取余后是否等于 0 来判断是否为整数。通过几行代码可以发现，当处理的值是""、[]、true、false 时，依然可以判断为是整数。这是非常严重的问题，这是因为在运算时，会进行类型转换，""、[]、false 都会被转换为 0，true 会被转换为 1，运算结果和值为数字时一样。关于类型转换的详细讲解，请参考核心技术-运算符一章。所以，使用该方法判断时，必须首先判断是否为数值类型。同时，值为数字构成的字符串时，因为类型转换的原因，依然会被正确判断。

第二种方法是利用Math.round等方法，判断取整后的数字是否和原始值相等，当然，如果原始值是字符串，永远会被判断判断为不相等,不在赘述。

第三种方法是利用parseInt判断取整后的数字是否和原始值相等，同样，如果原始值是字符串，永远会被判断判断为不相等。而且，当处理会被处理为科学计数的数字，也会被判断为不相等。

第四种方法是利用位运算判断取整后的数字是否和原始值相等，同样，如果原始值是字符串，永远会被判断判断为不相等。但是，位运算只能处理 32 位以内的数字，对于超过 32 位的无能为力。

第五种方法是利用 ES6 新增的方法Number.isInteger()判断数字是否为整数，是最为推荐的方法。当原始值为非数值类型的值时，会被判断为 false。同时可以发现，Number.isInteger(5.0)和 Number.isInteger(5)一样，也会被判断为整数，那是因为在 JavaScript 中，整数和浮点数是同样的存储方法，所以，5.0和5会被看做同一个值。

# 安全整数与任意精度整数

console.log("integer", Math.pow(2, 53)); // 9007199254740992
console.log("integer", Math.pow(-2, 53)); // -9007199254740992
console.log("integer", Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log("integer", Number.MIN_SAFE_INTEGER); // -9007199254740991
console.log(Number.isSafeInteger(Number.MAX_SAFE_INTEGER)); // true
console.log(Number.isSafeInteger(Number.MAX_SAFE_INTEGER + 1)); // false
console.log(Number.isSafeInteger(500)); // true
console.log(Number.MIN_SAFE_INTEGER <= 500 >= Number.MAX_SAFE_INTEGER); // false

console.log("integer", 9007199254740992); // 16位 9007199254740992
console.log("integer", 9007199254740993); // 16位 9007199254740992
console.log("integer", 90071992547409921); // 17位 90071992547409920
console.log("integer", 900719925474099212345); // 21位 900719925474099200000
console.log("integer", 9007199254740992123456); // 22位 9.007199254740992e+21

案例解析

在 JavaScript 中，只有一种数字类型 number，可以用来书写整数和浮点数，都会被存储为 10 的 64 位（8 比特）的浮点数，使用的是双精度格式，也就是 64 位二进制。这样就导致整数的精度只能到 53 个二进制位，也就是说，能够精确表示的整数范围是在 Math.pow(-2, 53)和 Math.pow(2, 53)之间，位数是 16 位，大于这个范围，无法精确表示。为了方便使用，ES6 引入了 Number.MAX_SAFE_INTEGER 和 Number.MIN_SAFE_INTEGER 表示这个精度范围的上下线，对比代码发现，不包括边界的数字 9007199254740992 和-9007199254740992。通过 ES6 的 Number.isSafeInteger()方法可以判断是否整数在安全范围，Number.MAX_SAFE_INTEGER + 1 后的 9007199254740992 已经不在安全整数范围内。如果用来判断一个整数是否在安全范围，除了使用 Number.isSafeInteger()，还可以结合使用 Number.MAX_SAFE_INTEGER 和 Number.MIN_SAFE_INTEGER 来手动判断。

接下来分析整数的位数，对于不适用小数点和指数计数法的整数，最多存储位数是 21 位。当位数大于 21 位，会被处理为指数计数法。当然，因为最大安全整数和最小安全整数的存在，超过 16 位，即使不会被处理为指数计数，后面数字也会被处理 0。而且，即使是 16 位，数值大于 Math.pow(2, 53)，也会被处理为和 Math.pow(2, 53)一样。需要特别注意的是，虽然 Math.pow(2, 53)被打印了出来，但并不是安全整数，这个必须明确。

# 浮点数运算误差处理

console.log("float", 0.1 + 0.2 === 0.3); // false
console.log("float", 0.1 + 0.2); // 0.30000000000000004
console.log("float", (0.1 * 10 + 0.2 * 10) / 10); // 0.3
console.log("float", 5.12 + 0.34 === 5.46); // true
console.log("float", 0.00005 + 0.002 === 0.00205); // true

console.log("Number.EPSILON", Number.EPSILON); // 2.220446049250313e-16
console.log("Number.EPSILON", Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON); // true
console.log("Number.EPSILON", Math.abs(0.0001 - 0.0002) < Number.EPSILON); // false

案例解析

在 JavaScript 中，小数的最大位数是 17，不过，浮点数的运算并不是 100%的精确，也就是说，二进制浮点数中的 0.1 和 0.2 本身就是不精确的，所以才会出现 0.1 + 0.2 !== 0.3 的情况。对于这种情况，如果要计算值，可以分别让浮点数 \*10，求和后再 /10 来获得正确结果。可以发现，其他情况下会进行精确的运算。

如果要判断两个浮点数是否相等，需要使用一个误差范围值，ES6 中，这个值被定义在了 Number.EPSILON 中，对应的值就是 2.220446049250313e-16。也就是说，如果 0.1 + 0.2 得到的 0.30000000000000004 减去 0.3，取绝对值后的误差值只要小于 Number.EPSILON，返回值是 true，就可以判断 0.1 + 0.2 === 0.3，因为误差在指定误差范围内。0.0001 - 0.0002 一行，返回时 false，说明 0.0001 和 0.0002 的误差范围超出了 Number.EPSILON 指定的误差范围，所以 0.0001 !== 0.0002。

# 判断是正数、负数还是零

console.log(Math.sign(5)); // 1
console.log(Math.sign(5.1)); // 1
console.log(Math.sign(-5)); // -1
console.log(Math.sign(0)); // 0
console.log(Math.sign(-0)); // -0
console.log(Math.sign("5")); // 1
console.log(Math.sign("-5")); // -1

案例解析

可以使用 ES6 提供的 Math.sign()判断一个数字是正数、负数还是零。返回 1，是正数，返回-1，是负数，返回 0，是 0，返回-0，是-0。如果判断的值是有数字构成的字符串，会先转换为数值类型在进行判断，处理结果没有差别。

说明

parseInt()和parseFloat()可以分别用ES6中的Number.parseInt()和Number.parseFloat()替代。ES6通过将全局方法移植到Number对象上，实现了方法的模块化。关于 parseInt()在处理 1000000000000000000000 和 0.0000006 这类极大或极小的数字时不能获得希望的结果，那是因为对于 JavaScript 来说，极大或极小的数字都会被处理为科学（指数）计数法形式，1000000000000000000000 处理为 1e+21，0.0000006 处理为 6e-7。

# 数值转字符串

const num = 5;
const num1 = 5.5;
console.log(num.toString(), num1.toString()); // 5 5.5
console.log(num.toLocaleString(), num1.toLocaleString()); // 5 5.5
const num2 = 5000.5;
console.log(num2.toString()); // 5000.5
console.log(num2.toLocaleString()); // 5,000.5

案例解析

在转换整数的位数小于 4 的情况下，toString()和 toLocaleString()的处理结果是一样的。在处理整数的位数大于等于 4 的情况，处理结果不同，toString()返回的是最原始的结果，形式上和数值类型是一样，只是转成了字符串类型。toLocaleString()处理结果在形式上发生了巨大变化，整数部分以 3 位数为一组，用逗号分隔。
